Cube Engine

Resumen

Este proyecto tiene dos partes. Por un lado, está un proyecto de biblioteca estática que tiene toda la lógica. Por el otro, está un proyecto que utiliza esa biblioteca para montar un escenario 3D con cubos y físicas.

Window y Scene

La clase que se encarga de generar la ventana utiliza como base este proyecto y esta modificado para incluir algunas funcionalidades extra (para que el proyecto pueda emitir sonidos). En esta clase también están los eventos de teclado, para la interacción del usuario.

La escena se encarga de crear todas las tareas necesarias para el funcionamiento del programa, por ejemplo, inicializar el sistema de físicas. Aparte de esto puede leer un archivo XML para crear una escena.

Kernel y Task

El kernel es el encargado de ejecutar todas las tareas, tiene una lista y las ejecuta en el orden en el que están en la lista.

Las tares heredan de una clase abstracta llamada Task. Hay cuatro tareas:

  • InputTask: se encarga de gestionar los eventos, como cerrar la ventana o el teclado, mandando mensajes.
  • MessageDispatcher: se encarga de procesar los mensajes.
  • PhysicsSystem: se encarga de hacer la simulación de las físicas, actualiza la posición y rotación de los objetos que no son estáticos. También comprueba las colisiones entre los objetos físicos.
  • RenderSystem: se encarga de renderizar los objetos, que se vean en escena.

Entity y Components

Una entidad es un objeto que está compuesto por componentes.

Hay muchos tipos de componentes, todos ellos heredan de una clase abstracta que se llama Component.

  • TransformComponent: el componente que representa la posición, rotación y escala de las entidades. Todas las entidades tienen este componente.
  • CameraComponent: este componente indica que la entidad es una cámara.
  • LightComponent: es el componente que representa la luz
  • RenderableComponent: es el componente que se muestra por pantalla, el modelo (un cubo).
  • PhysicsComponent: es el componente de físicas, con un rigidbody cubico. Tiene un id que sirve para detectar colisiones si se quiere realizar alguna acción ante estas
  • PlayerComponent: esta clase permite al usuario mover a un cubo con las teclas “WASD”, usando el sistema de físicas.

Mensajería

El sistema de mensajería está compuesto por tres clases principales:

  • Message: un mensaje está compuesto por su target, a quien va dirigido, y su acción, ya que un target puede realizar varias acciones. Por ejemplo, para los eventos que mueven al personaje jugable, para determinar la dirección a la que se debe aplicar la fuerza.
  • MessageObserver: es una clase abstracta que recibe un mensaje. Las clases que heredan de esta sobrescriben una función que recibe el mensaje para hacer distintas acciones.
  • MessageDispatcher: esta clase es la tarea que se encarga de recibir los mensajes y mandarlos a los distintos observer.

Hay varias clases que heredan del MessageObserver:

  • CloseMessage: se encarga de cerrar la ventana.
  • SoundMessage: se encarga de generar un sonido cuando el sistema de físicas detecta un impacto entre el jugador y una pared (gracias a la id).
  • PlayerComponent: este componente de la entidad recibe mensajes del InputTask para saber que teclas se han pulsado.
