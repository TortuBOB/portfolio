Aplicacion QT

Resumen

Este proyecto fue realizado para probar a hacer una herramienta con QT, que sirve para crear interfaces, y a crear y usar librerías dinámicas con C++.

La herramienta consiste en un editor de textos que exporta un XML, que podría ser usado posteriormente en un juego de tipo novela interactiva. Toda la lógica funciona con un proyecto de Visual Studio de biblioteca dinámica y la interfaz en otro proyecto con QT.

Modelo de datos (DLL)

El proyecto de la biblioteca dinámica está formado por cuatro clases que se encargan de toda la lógica del proyecto de QT:

  • Adventure: esta clase guarda una lista de diálogos. Tiene las funciones necesarias para añadir y eliminar diálogos, una función que devuelve el número de diálogos que tiene y otra que devuelve un diálogo si introduce el índice del diálogo en la lista.
  • Dialog: es la clase que representa un diálogo. Tiene un ID, que es un número para identificarlo. Aparte guarda otros datos como el título, el texto, el ID del próximo diálogo y una lista con las posibles respuestas. Tiene todas las funciones necesarias para modificar estos datos o para leerlos. En cuanto a las respuestas, tiene funciones para añadir y eliminarlas, y también otra que devuelve el número de respuestas que tiene y una que devuelve una respuesta si se introduce el índice de la respuesta en la lista.
  • DialogResponse: es la clase que representa las respuestas del diálogo. Tiene un texto y la ID del diálogo con el que conecta. Tiene las funciones necesarias para modificar estos datos o para leerlos.
  • Serializer: esta clase se encarga de guardar y cargar información de un XML. Tiene una función para guardar la aventura en la que se está trabajando, en el que se tiene que incluir la ruta donde se quiere guardar. También tiene la función para cargar, que necesita la ruta en el que se encuentra el XML. Esta función lee el XML para sacar los datos, crear una aventura con ellos y luego mostrarlo en la interfaz.

Interfaz (QT)

La interfaz se ha creado en el editor de QT, que viene con una ventana en la que se pueden colocar distintos elementos. La ventana de este proyecto está dividida en tres columnas.

  • En la columna izquierda está la lista de diálogos, donde se muestran todos los diálogos y se pueden seleccionar. En la parte de abajo hay dos botones, uno para añadir diálogos y otro para eliminar el que esté seleccionado en la lista.
  • En la columna central está la información del diálogo seleccionado, que se puede modificar. En la zona superior hay una etiqueta, que es texto que no se puede modificar, en la que pone “ID”, y al lado hay una caja de texto que solo acepta números, que es donde se puede modificar el ID del diálogo. Al lado de esta caja hay otra, que es el título del diálogo. Debajo hay una gran caja de texto, que es el contenido del diálogo. En la parte inferior hay una etiqueta que pone “Next ID” y una caja de texto que solo permite números y es para escribir cual es el ID del siguiente diálogo.
  • En la columna de la derecha hay una tabla para crear las posibles respuestas y debajo dos botones. La tabla tiene tres columnas: la primera solo muestra el número de la fila, la segunda es una caja de texto modificable, que es el contenido de la respuesta, y la tercera es el ID del diálogo con el que conecta la respuesta. Los botones sirven para añadir y eliminar respuestas, se elimina la fila que este seleccionada. La tabla se crea añadiendo los elementos con código.

Aparte de esto tiene una barra de herramientas, en la que únicamente está la opción “File”. Cuando se pulsa salen dos acciones, “Save” para guardar la aventura actual en un XML y “Load” para cargar una aventura de un archivo XML. Al pulsarlos pedirá una ruta para guardar o leer el archivo.

Para programar las distintas funciones que hacen los componentes de la interfaz se usa una función llamada “connect” que detecta distintas acciones en la interfaz y se puede especificar lo que hace como reacción. Los botones cuando son pulsados llaman a distintas funciones. La lista de los diálogos cambia la información del resto de la ventana cuando se selecciona otro diálogo. Los cuadros de texto actualizan su información cuando se cambia algo en ellos.
