3D Physics

Summary

This is the first project using the Bullet physics library. In this project a 3D scenario is set up with objects that simulate physics in different ways to learn how to use this library.

Entities and Sensors

An entity is an object made up of a rigid body and a 3D model. There are three types of entities: cubes, cylinders and spheres.

The physical part of the entity, the rigidbody, can be dynamic, which simulates physics (gravity, moves upon collision...), static, which cannot move, and kinematic, which does not simulate physics, but can move.

A sensor is an object that detects collisions, but unlike an entity, it can be passed through. Sensors are cube-shaped and can be invisible or have a 3D model.

There is a key that is a sensor, which is used to open a door. In this case it has a visible part (a model of a key) and the cube that detects collisions which is just the right size to occupy the whole model.

Car

A car is an object made up of several entities. The body is a cube and then it has four wheels which are cylinders. It also has two cylinders between the body and the front wheels, while the rear wheels are attached to the car directly. All these entities are dynamic.

The car can be controlled with "WASD". When the user presses the "W" key all the wheels start to rotate, and thanks to the friction with the ground the car moves. With "S" they rotate, but in the opposite direction to move backwards. The "A" and "D" keys turn the front cylinders that connect the body to the wheels, causing the wheels to turn, and so the car moves forward to the right or left.

Door and Platform

The door is composed of a dynamic cube entity and a static cylinder. Both are joined together and when the user takes the key, a motor that makes the door rotate on the cylinder is activated, opening it.

The platform, which is used to cross a hole, consists of a kinematic cube and an invisible sensor. The sensor is on top of the platform and when the car touches it, the platform crosses the hole.

Scene

The scene takes care of creating the camera and the light. Apart from this it generates all the scenery; the floor, the walls, and the rest of the objects.

The scene is also responsible for checking for collisions between sensors and the corresponding objects, and takes appropriate action when detected.

Libraries

SFML to create the window

Bullet Physics to simulate physics

OpenGl-Toolkit for the render system

GLM for mathematical calculations
