Cube Engine

Summary

This project has two parts. On the one hand, there is a static library project that has all the logic. On the other, there is a project that uses that library to assemble a 3D scenario with cubes and physics.

Window y Scene

The class that creates the window uses this project as a base and is modified to include some extra functionalities (for example, so that the project can emit sounds). In this class there are also the keyboard events, for user input.

The scene handles the creation of all the tasks necessary for the execution of the program, for example, initialising the physics system. Apart from this it can read an XML file to create a scene.

Kernel and Task

The kernel is in charge of executing all the tasks, it has a list and executes them in the order in which they are listed.

Tasks are an abstract class. There are four tasks:

  • InputTask: handles events, such as closing the window or keyboard inputs, by sending messages.
  • MessageDispatcher: is in charge of processing messages.
  • PhysicsSystem: is in charge of the physics simulation, updates the position and rotation of non-static objects. It also checks for collisions between physical objects.
  • RenderSystem: it is in charge of rendering the objects to be seen in the scene.

Entity y Components

An entity is an object that is made up of components.

There are many types of components, all of which inherit from an abstract class called Component.

  • TransformComponent: the component representing the position, rotation and scale of the entities. All entities have this component.
  • CameraComponent: this component indicates that the entity is a camera.
  • LightComponent: is the component that represents light
  • RenderableComponent: is the component displayed on the screen, the 3D model (a cube).
  • PhysicsComponent: this is the physics component, with a cubic rigidbody. It has an id that is used to detect any collisions in order to perform an action.
  • PlayerComponent: this class allows the user to move a cube with the "WASD" keys, using the physics system.

Messaging system

The messaging system has three main classes:

  • Message: a message is composed of its target, to whom it is aimed, and its action, as a target can perform several actions. For example, it is used to determine the direction in which the force should be applied in the player movement.
  • MessageObserver: is an abstract class that receives a message. Classes that inherit from it override a function that receives the message to do different actions.
  • MessageDispatcher: this class is the task in charge of receiving messages and sending them to the different observers.

There are several classes that inherit from the MessageObserver:

  • CloseMessage: is responsible for closing the window.
  • SoundMessage: it is in charge of producing a sound when the physics system detects an impact between the player and a wall.
  • PlayerComponent: this component of the entity receives messages from the InputTask to know which keys have been pressed.
