QT Application

Summary

This project was made to test how to make a tool with QT, which is used to create interfaces, and to create and use dynamic libraries with C++.

The tool consists of a text editor that exports an XML, which could be used later in an interactive novel type game. All the logic works with a Visual Studio dynamic library project and the interface in another project with QT.

Data model (DLL)

The dynamic-link library project consists of four classes that handle all the logic of the QT project:

  • Adventure: this class stores a list of dialogs. It has the necessary functions to add and remove dialogs, a function that returns the number of dialogs it has and a function that returns a dialog if you enter the index of it.
  • Dialog: is the class that represents a dialogue. It has an ID, which is a number to identify it. It also stores other data such as the title, the text, the ID of the next dialogue and a list of possible answers. It has all the necessary functions to modify this data or to read it. As for the answers, it has functions to add and delete them, and also one that returns the number of answers it has and one that returns an answer if the index of the answer is entered in the list.
  • DialogResponse: is the class that represents the responses of the dialog. It has a text and the ID of the dialog it connects to. It has the necessary functions to modify this data or to read it.
  • Serializer: this class is in charge of saving and loading XML files. It has a function to save the adventure you are working on, in which you have to include the path where you want to save it. It also has a function to load, which needs the path where the XML is located. This function reads the XML to extract the data, create an adventure with it and then display it in the interface.

Interface (QT)

The interface has been created in the QT editor, which comes with a window in which different elements can be placed. The window of this project is divided into three columns.

  • In the left column is the list of dialogues, where all dialogues are shown and can be selected. At the bottom there are two buttons, one to add dialogues and one to remove the selected dialogue from the list.
  • In the middle column is the information of the selected dialogue, which can be modified. In the upper area there is a label, which is text that cannot be modified, where it says "ID", and next to it there is a text box that only accepts numbers, which is where the ID of the dialogue can be modified. Next to this box is another box, which is the title of the dialogue. Underneath is a large text box, which is the content of the dialogue. At the bottom there is a label that says "Next ID" and a text box that only allows numbers, to type the ID of the next dialogue.
  • In the right column there is a table to create the possible answers and below it two buttons. The table has three columns: the first one shows only the row number, the second one is a modifiable text box, which is the content of the answer, and the third one is the ID of the dialogue to which the answer connects. The buttons are used to add and delete answers, the selected row is deleted. The table is created by adding the elements with code.

Apart from this it has a toolbar, where only the "File" option is available. When pressed, two actions appear, "Save" to save the current adventure to XML and "Load" to load an adventure from an XML file. Pressing them will request a path to save or read the file.

To program the different functions that the interface components do, a function called "connect" is used, which detects various actions on the interface and it can be specified what it does as a reaction. Each buttons when pressed call different functions. The list of dialogues changes the information in the rest of the window when another dialogue is selected. Text boxes update their information when something is changed in them.
